# JEWELFLAME AI CONTEXT
Last Updated: 2026-03-11

## Quick Reference
- **Engine**: Godot 4.6.1 Mono
- **Genre**: Turn-based strategy (Gemfire clone)
- **Active Scene**: Strategic layer (main_strategic.tscn)

## Architecture Map
```
Strategic Layer (main_strategic.tscn)
├── GameState (autoload) - Runtime data
├── EventBus (autoload) - Global signals  
├── StrategicLayer (Node2D) - Hex map rendering
├── StrategicMenuPanel (PanelContainer) - Left UI (YOU ARE HERE)
└── AIController (autoload) - Opponent turns
```

## Data Models (Read-Only)
- ProvinceData: id, name, owner_id, governor_id, gold, food, soldiers, loyalty, protection, terrain_type
- FamilyData: id, name, color, ai_personality
- CharacterData: id, name, is_lord, is_ruler, command, leadership, charm

## Critical Constraints (DO NOT BREAK)
1. Province IDs are 1-5 only (Dunmoor, Carveti, Cobrige, Banshea, Petaria)
2. Family IDs: "blanche", "lyle", "coryll" only
3. owner_id (not family_id) links provinces to families
4. governor_id (not lord_id) links provinces to characters
5. EventBus signals must maintain current signatures

## Current Pain Points
- Portrait system uses folder scanning, not portrait_path field
- Node paths are deep and brittle ($MarginContainer/ContentStack/...)
- Missing type hints in function signatures
- No debug visualization mode
